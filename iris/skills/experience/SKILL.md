# Experience Library

Your long-term memory of what worked, what failed, and why.

## Before every analysis
The system automatically injects relevant memories at the start of each session.
If you need memories from a different angle, call `recall` with a specific context.
If Warning Zone experiences appear, address them in your reasoning.

## Saving knowledge
Use `remember` to save anything worth keeping:
- Observations: facts you extracted from data
- Experiences: lessons learned (with zone: golden/warning and methodology)
- Notes: analysis summaries and conclusions
- Predictions: automatically saved when you run build_dcf

## Experience dedup
The system automatically deduplicates experiences using semantic similarity.
Similar entries get merged; near-duplicates increment evidence count.

## Boundaries
- Factual and pattern experiences: saved autonomously
- Strategic experiences: you propose, human confirms
- Soul philosophy, DCF formulas, Harness parameters: never touch
